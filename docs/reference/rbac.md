# RBAC Reference

The operator follows least-privilege principles. It only requests the permissions it strictly needs for the current phase of implementation.

---

## ClusterRole — rca-operator-manager-role

Generated by `make manifests` from `// +kubebuilder:rbac:` markers in the controller source. Do **not** edit `config/rbac/role.yaml` by hand.

### Current Permissions (Phase 1)

| API Group | Resources | Verbs | Purpose |
|---|---|---|---|
| `rca.rca-operator.io` | `rcaagents` | get, list, watch, create, update, patch, delete | Reconcile RCAAgent CRs |
| `rca.rca-operator.io` | `rcaagents/status` | get, update, patch | Write status conditions |
| `rca.rca-operator.io` | `rcaagents/finalizers` | update | Manage the cleanup finalizer |
| `rca.rca-operator.io` | `incidentreports` | get, list, watch, create, update, patch, delete | Create and resolve incidents |
| `rca.rca-operator.io` | `incidentreports/status` | get, update, patch | Write incident status |
| `rca.rca-operator.io` | `incidentreports/finalizers` | update | Manage incident finalizers |
| `""` (core) | `pods` | get, list, watch | Pod watcher informer |
| `""` (core) | `namespaces` | get, list, watch | Validate watchNamespaces |
| `""` (core) | `secrets` | get, list, watch | Validate aiProviderConfig.secretRef |
| `""` (core) | `events` | create, patch | Emit Kubernetes events |
| `coordination.k8s.io` | `leases` | get, list, watch, create, update, patch, delete | Leader election |

### Planned Permissions (Phase 2+)

These will be added as features are implemented:

| API Group | Resources | Verbs | Purpose |
|---|---|---|---|
| `""` (core) | `pods/log` | get | Evidence gathering — fetch container logs |
| `""` (core) | `nodes`, `endpoints`, `services` | get, list, watch | Node and service health watching |
| `apps` | `deployments`, `replicasets`, `statefulsets` | get, list, watch | Correlate incidents with deploys |
| `apps` | `deployments` | update, patch | Remediation — rollback (autonomy ≥ 2 only) |
| `metrics.k8s.io` | `pods`, `nodes` | get, list | Metrics-based correlation |

> **Note:** Write permissions on Deployments are only used at autonomy level ≥ 2. For observe/suggest-only deployments you can restrict the role to `["get", "list", "watch"]`.

## ServiceAccount

The operator runs as `rca-operator-controller-manager` in the `rca-operator-system` namespace.

```bash
kubectl get serviceaccount -n rca-operator-system
kubectl get clusterrolebinding | grep rca-operator
```

## Viewing the Live Role

```bash
# See what the operator is currently allowed to do
kubectl get clusterrole rca-operator-manager-role -o yaml
```
