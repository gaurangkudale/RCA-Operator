# Dashboard API Reference

The RCA Operator exposes an HTTP API on port 9090 (configurable via `--dashboard-bind-address`). All endpoints return JSON unless noted otherwise.

## Base URL

```
http://<operator-host>:9090
```

---

## Incidents

### GET /api/incidents

Returns all `IncidentReport` CRs visible to the operator.

**Query Parameters:**

| Parameter | Type | Description |
|---|---|---|
| `namespace` | string | Filter by namespace |
| `phase` | string | Filter by phase: `Detecting`, `Active`, `Resolved` |
| `severity` | string | Filter by severity: `P1`, `P2`, `P3`, `P4` |
| `type` | string | Filter by incident type (e.g. `CrashLoopBackOff`) |
| `query` | string | Free-text search across summary, type, and namespace |
| `limit` | int | Max results to return (default: 100) |
| `offset` | int | Pagination offset |
| `sort` | string | Sort field. Options: `severity`, `phase`, `firstObservedAt` (default) |

**Response:**

```json
{
  "items": [
    {
      "name": "crashloopbackoff-payment-abc123",
      "namespace": "production",
      "phase": "Active",
      "severity": "P2",
      "incidentType": "CrashLoopBackOff",
      "summary": "CrashLoopBackOff: container app in pod payment-abc123 (restarts: 8)",
      "firstObservedAt": "2026-04-01T10:00:00Z",
      "activeAt": "2026-04-01T10:05:00Z",
      "lastObservedAt": "2026-04-01T10:15:00Z",
      "notified": true,
      "signalCount": 5,
      "relatedTraces": ["abc123def456"],
      "blastRadius": ["api-gateway", "frontend"],
      "rca": {
        "rootCause": "Memory leak in payment-svc",
        "confidence": "0.92",
        "playbook": ["kubectl rollout undo ..."],
        "evidence": ["Trace abc123: 500ms latency spike"],
        "investigatedAt": "2026-04-01T10:20:00Z"
      }
    }
  ],
  "total": 42
}
```

## AI Investigation

### GET /api/investigate/{ns}/{name}

Returns the existing AI RCA result for an incident. Returns `404` if no investigation has been run.

**Response:**

```json
{
  "rootCause": "Memory leak in payment-svc caused OOMKilled events",
  "confidence": "0.92",
  "playbook": [
    "kubectl rollout undo deployment/payment-svc -n production",
    "kubectl scale deployment/payment-svc --replicas=6"
  ],
  "evidence": [
    "Trace abc123: 500ms latency spike on /checkout",
    "Log: java.lang.OutOfMemoryError at 10:15:01"
  ],
  "investigatedAt": "2026-04-01T10:20:00Z"
}
```

### POST /api/investigate/{ns}/{name}

Triggers an AI-powered investigation for the specified incident. Returns the RCA result when complete.

The operator assembles context (incident metadata, K8s events, related traces, metric anomalies, error logs), redacts PII, and submits it to the configured LLM endpoint.

**Request body:** None required.

**Response:** Same as `GET /api/investigate/{ns}/{name}`.

**Error responses:**

| Status | Reason |
|---|---|
| `404` | Incident not found |
| `503` | AI endpoint not configured or unavailable |

## Server-Sent Events

Both SSE endpoints keep the connection open and push events as they occur. Keepalive comments (`: keepalive`) are sent every 30 seconds.

### SSE /api/stream/correlation

Live signal stream for the Live Correlation Stream bar.

**Events:**

| Event Type | Description |
|---|---|
| `connected` | Emitted once on connect. Payload: `{"channel":"correlation","time":"..."}` |
| `signal.new` | New signal received. Payload: `{"type":"...","service":"...","message":"...","source":"prometheus\|jaeger\|signoz\|rca-operator\|ai","time":"..."}` |
| `incident.created` | New incident created. Payload: `{"name":"...","namespace":"...","type":"...","severity":"...","time":"..."}` |

### SSE /api/stream/topology

Live topology graph updates.

**Events:**

| Event Type | Description |
|---|---|
| `connected` | Emitted once on connect. Payload: `{"channel":"topology","time":"..."}` |
| `topology.update` | Full ServiceGraph JSON (same shape as `GET /api/topology`) |
